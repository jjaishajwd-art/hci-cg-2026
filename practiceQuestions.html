Part C: Short Answer
  • Explain the difference between a semantic element (e.g., nav) and a generic element (e.g., div).

    Both look similar visually, but <nav> tells assistive tech: ‚ÄúThis is a navigation region.‚Äù Conveys meaning/purpose of content to browsers, developers, and assistive tech. Helps screen readers and search engines understand structure.Improves readability and maintainability of code.
    <div> is just a container; screen readers won‚Äôt know its purpose without extra ARIA attributes. Has no inherent meaning; just a container for styling or layout.Screen readers treat it as just a block of content, Styled only visually; meaning must be inferred by developer and does not indicate content purpose.

  • Why does KLM not model high-level planning and interpretation?

    Keystroke-Level Model (KLM)

    KLM (Keystroke-Level Model) focuses only on low-level, observable actions of expert users. It predicts execution time based on keystrokes, clicks, and pointing movements and It ignores cognitive tasks like thinking, planning, or problem-solving, because these are highly variable and hard to measure consistently.
    In other words, KLM assumes the user already knows what to do, so it‚Äôs about speed of doing, not deciding.

    Example: Filling a form

    User has to fill a form:

    • KLM measures: typing each letter, pressing tab, clicking submit.
    • KLM ignores: figuring out which field to fill first, or reading instructions carefully.
